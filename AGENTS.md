# KetchupTimer Agent Guidelines

このファイルは、KetchupTimerプロジェクトに参加するAIエージェント向けのプロジェクト概要と基本ガイドラインをまとめたものです。

## 1. プロジェクト概要
KetchupTimerは、ポモドーロ・テクニックを用いたChrome拡張機能です。
「ケチャップボトル」をコンセプトとし、作業中（Focus）にケチャップが減り、休憩中（Break）に補充されるという視覚的フィードバックを提供します。

詳細は `.context/spec.md` を参照してください。

## 2. 基本ガイドライン

### 仕様の参照と更新
- 実装を開始する前に、必ず `.context/spec.md` を読み込み、最新の仕様とディレクトリ構成を把握してください。
- 機能の追加や変更を行った際は、`.context/spec.md` の該当箇所を適切に更新してください。

### スキルの活用
- `.agents/skills/` 配下には、このプロジェクト固有の技術スタックや設計方針が定義されています。
  - `chrome-extension`: Manifest V3準拠の設計、Service Worker、状態管理などのガイドライン。
  - `design`: ケチャップコンセプトに基づいた視覚的フィードバックやUI/UXの方針。
  - `typescript`: プロジェクトの型定義やコードスタイルの方針。
- 各スキルに記載された指示は、プロジェクトの技術的整合性を保つための「必須事項」として扱ってください。

### ワークフローの遵守
- `.agents/workflows/` 配下には、開発、テスト、リリースの手順が定義されています。
  - `development.md`: 開発環境のセットアップから動作確認までの流れ。
- プロセスの逸脱を避け、定義されたステップに従って作業を進めてください。

## 3. 開発上の注意点
- **ステートレスな設計**: Background script (Service Worker) は常に `chrome.storage` を真実のソースとして扱い、状態を持たないようにしてください。
- **動的アイコン**: ケチャップの減少・補充を表現するアイコン描画ロジックは、このプロジェクトのコア機能です。
- **後方互換性と安全性**: 既存のユーザーデータを壊さないよう、ストレージのスキーマ変更には慎重に対応してください。

これらのガイドラインに従い、一貫性のある高品質なコードを提供してください。
